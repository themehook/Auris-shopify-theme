## How to use

1. Before editing any section, open the matching skill file first.
2. Read the **Requirements**, **Design spec**, and **Schema** sections to understand the intent.
3. Check **Known decisions** for things that look odd but are intentional.
4. After a significant change, update the skill file so the context stays current.
